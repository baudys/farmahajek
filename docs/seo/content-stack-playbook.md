# Content SEO Stack Playbook

## Content Model Requirements

Each new page/post must define:

1. Primary keyword and intent.
2. Title tag and meta description.
3. H1 aligned with title intent.
4. Canonical URL.
5. Internal links:
- at least one link to service/commercial page,
- at least one link to a related blog page.
6. Schema template:
- `Article` for blog content,
- `FAQPage` when FAQ block is present,
- `BreadcrumbList` for deep pages.

## Internal Linking Rules

1. Use descriptive anchor text (no generic “click here”).
2. Link from new posts to:
- `/cs/sluzby`,
- `/cs/vydejni-mista`,
- at least one related post.
3. Update older cornerstone pages with links to new posts.

## Publishing Checklist

1. Route-level metadata exists.
2. Canonical and hreflang are valid.
3. Schema validates in Rich Results Test.
4. Headings follow strict hierarchy.
5. Images use descriptive alt text and compressed formats.
6. Page passes local Lighthouse SEO checks.
